Reconciliação vs. implementado

MigrationBanner · CatchUpBanner · MigrationFlow · MigrationCard — deltas vão a Correct Course
Preservado do contrato atual
  • Detecção por pendências (não dia específico); ordem mês → semana → dia mantém
  • Uma decisão explícita por tarefa; destinos today/week/month/future/cancel via /migrate/ mantém
  • Atalhos de teclado e Enter na ação nomeada mantém
  • Sem toast de sucesso; o banner some quando a fila zera mantém
  • Progresso "{i} de {n} revisadas" com aria-live mantém
Amplia o contrato — requer Correct Course / arquitetura
  • Unificação: 1 banner + 1 fluxo no lugar de MigrationBanner + CatchUpBanner separados; contagem por fonte delta
  • Reusa a estrutura do ritual (fontes/decisões/contexto), dentro do shell — hoje é um Dialog card-a-card. Revisa o DESIGN.md, que dizia "tela cheia / Full-screen Flow" delta
  • Seletor de destino rico reusado (hoje são mini-campos inline no card) delta
  • Contexto com progresso + decidido + restantes por fonte (adapta o rail do ritual; sem calendário-alvo) delta
  • Pausar explícito + retomar (hoje fechar reinicia índice 0, sem persistir posição) delta
  • Resumo factual ao fim (hoje o Dialog fecha seco) delta
  • Erro de escrita no fluxo (hoje só no TaskDestinationDialog) delta
Decisões fechadas
  1. Fila unificada: sim — arquitetura mescla /migration/queue/ + /catch-up/queue/ numa lista ordenada com rótulo/contagem por fonte.
  2. Ação enfatizada: "Migrar para hoje" é a ação em destaque em toda fonte (inclusive itens de mês).
  3. "Concluir" no fluxo: não — só migrar / adiar (escolher destino) / cancelar. Migração não conclui tarefa.
  4. Retomar: basta os itens restantes (decisões persistem por item); sem persistir posição/fonte exata.
